# Redesign Context

## Audience

This site needs to be legible to three groups:

1. **Potential collaborators** — researchers, engineers, protocol teams who might work with River Computer
2. **Investors** — people evaluating the work as a bet
3. **People trying to understand the arc** — how this work evolved out of Regen Network and where it's going

## Core Identity

River Computer is a **software and protocol design studio** for the development of **coordination surfaces between people, living systems, and machines**.

### On "coordination surfaces"

The word *surface* is deliberate. It borrows from the "coordination interface" language developed in the Waka corpus but reaches further:

- A *surface* implies something you can touch, work on, and build upon — not just a screen or an API
- It carries the sense of a shared working plane where different kinds of actors (human, ecological, computational) meet
- "Coordination interface" is precise but technical; "coordination surface" is more spatial and inviting — it suggests a table, a deck, a shared ground
- The risk: it's not immediately obvious what it means. That's OK if the rest of the site grounds it. But the first pass should test whether we can make "surface" land without a glossary

## Relationship to Regen Network

The site should make legible that this work **evolved out of** Regen Network — not as a departure but as a continuation and deepening. The specifics of how to frame that are still open.

## Relationship to Waka

Waka is the primary prototype and the claims engine / coordination interface that River Computer is building. The site should reference Waka but River Computer is the studio; Waka is the product. The site is for River Computer, not for Waka (Waka has its own README and positioning docs).
